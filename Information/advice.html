Tips på vad man kan gör för att öka tillgängligheten för alla

När man utvecklar webbsidor är det viktigt att tidigt tänka på tillgängligheten för alla. Det är lättare att lösa problem om man upptäcker dem tidigt. Det finns verktyg som kan hjälpa utvecklare att utvärdera dessa problem men de kan inte avgöra om din webbsida är tillämpad för alla. För det behöver man kunniga och medvetna utvecklare.

Aria

Aria (Accessible Rich Internet Applications) är ett set av attribut som definierar tillvägagångssätt för att göra webbinnehåll och webbapplikationer mera tillgängliga för människor som har olika typer av funktionsnedsättningar. Det kompletterar HTML så att interaktioner och widgets som vanligtvis används i applikationer kan överföras till Assistive Technologies när det inte annars är en mekanism. Till exempel gör ARIA tillgängliga navigation märken i HTML4, JavaScript-widgets, formulär tips och felmeddelanden, uppdateringar av levande innehåll och mer.

ARIA tillhandahåller attribut för att deklarera det aktuella tillståndet för en gränssnitt-widget. Exempel inkluderar (men är visserligen inte begränsat till):

  • aria-checked: anger status för en kryssruta eller radioknapp
  • aria-disabled: indikerar att ett element är synligt men inte redigerbart eller på annat sätt kan användas
  • aria-grabbed: anger objektets "grepp" -status i en drag-och-släpp-operation

Synskadade/synnedsättning

  • Se till att all ljud och bild är enkel att se och höra
  • Skriv tydliga rubriker
  • Tydliga länkar som beskriver vart länken leder
  • Har du filmer eller videos på din webbsida är det viktigt att det finns undertext, som även är beskrivande
  • Skriv korrekt alt text på dina bilder

Hörselskadade/Hörselnedsättning

  • Har du filmer eller videos på din webbsida är det viktigt att det finns undertext
  • Använd teckenspråk som ett tillägg
  • Se till att allt på din webbsida är enkelt att se
  • Se till att all ljud och bild är enkel att se och höra

Utdaterade webbläsare

  • Se till att din kod är anpassad för alla webbläsare
  • En bra sida att använda är https://caniuse.com/

Responsiv

  • Anpassad för alla enheter.
  • Här kan man använda media queries, bootstrap etc.

Koncentrationssvårigheter

  • Lugna sidor med lite distraktioner
  • Rörliga bilder och annonser är jobbigt

Epilepsi

  • Alltid varna innan man kommer in på sidan om det kan finnas något som kan trigga epilepsi
  • Försök att undvika blinkande design
